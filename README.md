# IOS Test Task - SHIFT Lab

## Стек и Технологии
- **Язык:** Swift
- **UI Framework:** SwiftUI
- **Минимальная версия iOS:** 15.0
- **Архитектура:** MVVM (Model-View-ViewModel)
- **Принципы:** SOLID, Clean Architecture
- **Хранение данных:** UserDefaults
- **Сеть:** URLSession + async/await
- **API:** [FakeStoreAPI](https://fakestoreapi.com/products)

## Архитектура проекта
Проект следует принципам **Clean Architecture** и **MVVM**:
- **Models** — чистые структуры данных (`Product`)
- **Services** — сетевой слой и работа с хранилищем (протоколы + реализации)
- **ViewModels** — бизнес-логика, валидация, состояние экранов
- **Views** — SwiftUI представления

**SOLID:** Использован принцип Dependency Inversion. ViewModel зависит от протоколов (`StorageServiceProtocol`, `NetworkServiceProtocol`), а не от конкретных реализаций, что позволяет легко писать Unit-тесты.

## Реализованный функционал

### Экран 1: Регистрация
- Поля: Имя, Фамилия, Дата рождения, Пароль, Подтверждение пароля.
- **Валидация в реальном времени:** 
  - Имя/Фамилия: минимум 2 символа, разрешены только буквы и пробелы.
  - Возраст: минимум 15 лет.
  - Пароль: минимум 8 символов, наличие цифры и заглавной буквы.
  - Пароли должны совпадать.
- Ошибки валидации отображаются красным текстом под соответствующим полем.
- Кнопка «Зарегистрироваться» неактивна, пока форма не валидна.
- Кэширование имени и статуса регистрации в `UserDefaults`.

### Экран 2: Главный экран
- Динамический список товаров, получаемый с API (название, описание, цена, изображение).
- Кнопка «Приветствие» находится в навбаре справа, открывающая модальное окно с именем пользователя из кэша.
- Обработка состояний: загрузка (`ProgressView`), ошибка (с кнопкой повтора), пустой список.
- Переключатель темы (Dark/Light Mode) находится в навбаре слева.
- Полная адаптация UI под Dark Mode.

### Сохранение сессии
При повторном запуске приложения, если пользователь уже зарегистрирован, экран регистрации пропускается и сразу открывается Главный экран.

### Unit-тесты:
Покрыта логика валидации `RegistrationViewModel` с использованием Mock-объектов.

## Как запустить?
### 1. Клонируйте репозиторий
git clone https://github.com/MatveyMakarenko/shiftLab_IOS.git
### 2. Перейдите в папку проекта
cd shiftLab_IOS
### 3. Откройте проект в Xcode
open shiftLabTest.xcodeproj
### 4. Выберите устройство для запуска
В верхней панели Xcode выберите симулятор (например, iPhone 16 pro)
### 5. Запустите проект
Нажмите кнопку Run в левом верхнем углу Xcode (или сочетание клавиш CMD + R)
